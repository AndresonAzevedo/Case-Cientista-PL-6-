Q.2.2: Qual o modelo preditivo voc utilizaria para traar uma estratgia objetivando o aumento de adeso de acordos (Descreva a tcnica utilizada)

Modelo de regresso logistica

  • um modelo probabilistico que permite a probabilidade associada ocorrncia de um determinado evento dado os valores observados de outras varveis;

  • Esta tcnica costuma ser aplicada em marketing, deteco de fraudes, em riscos financeiros e seguros;

Exemplos: Identificao de transaes fraudulentas, identificar usurios com maior propenso responder um contato de marketing. Para o case tcnico: Identificar quais clientes em atraso tem maior propenso a aderir a um acordo

  • Entre as suas principais vantagens de uso, podemos destacar:

  1. Facilidade de lidar com varaveis categricas independentes;

  2. Fornece resultados em termos de probabilidade;

  3. Facilidade de classificar indivduos em categorias;

  4. Necessita de um nmero pequenos de suposies;

  5. um modelo de fcil interpretao, determina o efeito que os coeficientes exercem sobre a chance de um evento ocorrer. Se um coeficiente estimado:

  • Positivo, Aumenta a probabilidade;

  • Negativo, Diminui a probabilidade.

  1. Como o modelo retorna probabilidades, necessrio a definir uma regra de predio, em geral se utiliza 0.5, ou seja:
  • Se \(P(Y = 1) > 0,5\), ento se classifica Y = 1;
  • Se \(P(Y = 1) < 0,5\), ento se classifica Y = 0.

Contudo, para a definio da tcnica, deve sempre se considerar o tipo de estudo, problema e contexto.

Q.2.3: Quais indicadores e ferramentas voc utilizaria para avaliar a perfomance/aderncia desse modelo? (Descreva os indicadores utilizada)

  • Para uma boa estimativa da eficincia do modelo em termos de classificao, recomendado separar a amostra em duas partes:

  1. uma parte para estimao do modelo (Conjunto de treinamento), e

  2. outra parte para testar a eficincia da classificao (holdolt sample)

Para avaliao do poder de descriminao do modelo, se utiliza a seuinte tabela, chamada de matriz de confuso.

Valor Observado
Y = 1 Y = 0
Valor Estimado \(\hat{Y} = 1\) VP FP
\(\hat{Y} = 0\) FN VN

A partir da tabela acima possvel calcular as mtricas de avaliao, para reduzir trabalho de escrita, vou coloca-ls j em termos do problema proposto na questo seguinte

  • Sensibilidade: a probabildade de classificar um usurio que vai fazer acordo quando ele realmente faz acordo. Ou seja, dos usurios que fizeram acordo quantos foram corretamente classificados pelo modelo ?

\[S = \dfrac{VP}{VP+FN}\]

  • Especificidade: a probabilidade de classificar um usurio que no vai fazer acordo quando ele realmente no fez acordo. Ou seja, dos que no fizeram acordo, quantos foram corretamente classificados?

\[E = \dfrac{VN}{VN+FP}\]

  • Valor preditivo positivo (Precision): Dos usurios classificados como provvel acordo quantos foram corretamente identificados ?

\[VPP = \dfrac{VN}{VN+FP}\]

  • Valor Preditivo Negativo: Dos usurios classificados como no acordo, quantos foram corretamente identificados ?

\[VPN = \dfrac{VN}{VN+FN}\]

  • Preciso (Acurcia): a proporo de usurios que foram corretamente classificados ou como acordo ou como no acordo.

\[ACC = \dfrac{VP+VN}{VP+VN+FP+FN}\] Geralmente, sensibilidade alta, significa que estamos classificando muitos usurios como provvel acordo. Nessa situao a especificidade baixa, pois so mtricas opostas.

A escolha do modelo ento, vai ser feita por meio das mtricas acima, em especial sensibilidade e especficidade, em geral a importncia dessas mtricas depende do problema, em alguns problemas importante ter um sensibilidade maior, em outros a especificidade. A definio de qual das mtricas mais importante vai depender do custo associado ao erro em cada uma das situaes. Para uma definio razovel para o problema necessrio entender com a pessoa de negcio. Para nosso problema, poderia assumir a sensibilidade mais importante, porque se a sensibilidade for baixa, estou deixando de identificar usurios que iriam aderir o acordo, o que traria uma perda de arrecadao. Mas vale ressaltar que poderia estar correndo risco, visto que no tenho ideia dos custos associados a uma especificidade baixa. Ento serei cauteloso, vou buscar a regra de classificao que maximiza a sensibilidade + especificidade.

perform_fn <- function(cutoff) 
{
  predicted_churn <- ifelse(pred >= cutoff, "1", "0")
  conf <- confusionMatrix(table(predicted_churn, acordo_train$RESPOSTA))
  accuray <- conf$overall[1]
  sensitivity <- conf$byClass[1]
  specificity <- conf$byClass[2]
  out <- t(as.matrix(c(sensitivity, specificity, accuray))) 
  colnames(out) <- c("sensitivity", "specificity", "accuracy")
  return(out)
}
